# PEB-SRV Backend Integration Guide

This guide describes how to connect the **NEXUS PEB (Persistent Engineering Brain) Observability UI** to a live instance of the `peb-srv` backend daemon on port `4206`.

> **Source of truth:** This document reflects the actual `peb-srv` REST API as implemented in `nexus/typescript/peb-srv/src/routes/`. See also the project README at `nexus/typescript/peb-srv/README.md`.

---

## 1. Quick Overview & Architecture

The NEXUS Observability Dashboard visualizes real-time governance events, causal lineage, capability enforcement, circuit-breaker state, and state diffs generated by AI agent execution pipelines.

By default, the application runs with an **In-Memory Mock Scheme Engine**. You can switch to **LIVE API Mode** at runtime via the UI or by configuring environment defaults.

### Key Connection Settings

| Parameter | Default Value | Description |
| :--- | :--- | :--- |
| **Api Mode** | `mock` (or `live`) | Toggles client mode between mock generator & REST/SSE endpoints |
| **Base URL** | `http://localhost:4206` | Target URL of the running `peb-srv` REST API service |
| **SSE Path** | `/api/peb/events/stream` | Server-Sent Events endpoint for real-time log tailing |

To set the base URL programmatically, edit `.env` or set `VITE_PEB_API_BASE_URL`:

```env
VITE_PEB_API_BASE_URL=http://localhost:4206
```

## 2. CORS & Protocol Requirements

Your `peb-srv` service should accept cross-origin requests from the frontend origin.

### Required HTTP Headers
```http
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, OPTIONS
Access-Control-Allow-Headers: Content-Type, Authorization
```

### Server-Sent Events (SSE) Header Configuration
For `/api/peb/events/stream`:
```http
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
X-Accel-Buffering: no
```

## 3. Complete REST & SSE API Endpoint Specifications

All endpoints are mounted under `/api/peb` except the root `/health` probe.

### 3A. Health & System Rollups

#### `GET /health`
Returns overall system status and record totals across all PEB tables.
* **Response `200 OK`**:
```json
{
  "status": "healthy",
  "counts": {
    "event_count": 18420,
    "transaction_count": 5100,
    "violation_count": 32,
    "decision_count": 1200,
    "trace_count": 8900,
    "circuit_breakers_tripped": 2
  }
}
```

#### `GET /api/peb/health/circuit-breakers`
Returns safety circuit breaker states across all agent roles, tripped-first.
* **Response `200 OK`**:
```json
{
  "circuit_breakers": [
    {
      "role": "db_migration_agent",
      "tripped": 3,
      "tripped_at": "2026-07-22T16:45:00.000Z",
      "retry_after": 3600,
      "error": "3 consecutive violations",
      "failure_count": 3,
      "updated_at": "2026-07-22T16:45:00.000Z",
      "state": "OPEN"
    }
  ]
}
```
**`state` values:** `OPEN` (tripped, within cooldown), `RECOVERING` (tripped > 0, past cooldown), `CLOSED` (not tripped).

#### `GET /api/peb/health/violations/summary?window=24h&group_by=severity`
Returns rolling window violation breakdown.
* **Query Params**:
  * `window`: `1h` | `24h` | `7d` | `30d` (default: all time)
  * `group_by`: `severity` | `violation_type` | `entity_id` (default: `severity`)
* **Response `200 OK`**:
```json
{
  "group_by": "severity",
  "window": "24h",
  "summary": [
    { "key": "CRITICAL", "total": 14, "resolved_total": 10 },
    { "key": "WARNING", "total": 12, "resolved_total": 8 },
    { "key": "FATAL", "total": 6, "resolved_total": 2 }
  ]
}
```

#### `GET /api/peb/health/entropy?group_by=entropy_class&window=14d`
Returns decision entropy rollup grouped by the requested dimension, plus per-day trend.
* **Query Params**:
  * `group_by`: `entropy_class` | `author_id` | `status` (default: `entropy_class`)
  * `window`: `1h` | `24h` | `7d` | `30d`
* **Response `200 OK`**:
```json
{
  "group_by": "entropy_class",
  "window": "14d",
  "summary": [
    { "key": "ELEVATED", "total": 45, "last_seen": "2026-07-22T17:00:00Z", "first_seen": "2026-07-08T09:00:00Z" },
    { "key": "LOW", "total": 30, "last_seen": "2026-07-21T12:00:00Z", "first_seen": "2026-07-08T08:00:00Z" }
  ],
  "trend": [
    { "day": "2026-07-21T00:00:00Z", "key": "ELEVATED", "total": 5 },
    { "day": "2026-07-22T00:00:00Z", "key": "ELEVATED", "total": 8 }
  ]
}
```

### 3B. Event Ledger & SSE Stream

#### `GET /api/peb/events`
Cursor-paginated audit trail. Uses `since` (exclusive `governance_events.id`) for efficient tailing.
* **Query Params**:
  * `since` (integer): Filter events with `id > since` (exclusive)
  * `event_type` (string): `admission` | `decision` | `violation` | `circuit_breaker_trip` | `replay`
  * `plan_id` (string): Exact match filter
  * `agent_role` (string): Exact match filter
  * `work_request_id` (string): Exact match filter
  * `limit` (integer): Page size (default: `20`, max: `500`)
  * `offset` (integer): Pagination offset (default: `0`)
* **Response `200 OK`**:
```json
{
  "events": [
    {
      "id": 1042,
      "receipt_id": "rcpt_889201_violation",
      "event_type": "violation",
      "plan_id": "plan_nexus_002",
      "agent_role": "db_migration_agent",
      "work_request_id": "wrk_904",
      "payload": {
        "status": "DENIED",
        "violation_type": "UNAUTHORIZED_CAPABILITY_ACCESS",
        "latency_ms": 42
      },
      "replayed_at": null,
      "created_at": "2026-07-22T17:15:30.000Z"
    }
  ],
  "next_cursor": 1042,
  "limit": 20,
  "offset": 0
}
```
**`next_cursor`** is the `id` of the last event in the page, or `null` if the page has fewer than `limit` items (end of data).

#### `GET /api/peb/events/{receipt_id}`
Returns a single governance event by receipt ID.
* **Response `200 OK`**:
```json
{
  "event": {
    "id": 1042,
    "receipt_id": "rcpt_889201_violation",
    "event_type": "violation",
    "plan_id": "plan_nexus_002",
    "agent_role": "db_migration_agent",
    "work_request_id": "wrk_904",
    "payload": { "status": "DENIED" },
    "replayed_at": null,
    "created_at": "2026-07-22T17:15:30.000Z"
  }
}
```
* **`404`**: Returns `{ "error": "event not found" }`.

#### `POST /api/peb/events/{receipt_id}/replay`
Stamps `replayed_at = now()` on the event (idempotent) and pushes a `replay` event on the SSE bus.
* **Response `200 OK`**:
```json
{
  "replayed": {
    "id": 1042,
    "receipt_id": "rcpt_889201_violation",
    "event_type": "violation",
    "work_request_id": "wrk_904",
    "plan_id": "plan_nexus_002",
    "agent_role": "db_migration_agent",
    "payload": { "status": "DENIED" },
    "created_at": "2026-07-22T17:15:30.000Z",
    "replayed_at": "2026-07-22T17:20:00.000Z"
  }
}
```
* **`404`**: Returns `{ "error": "event not found" }`.

#### `GET /api/peb/events/stream?plan_id=&agent_role=`
**Server-Sent Events (SSE)** live log stream. Polls `governance_events` every 1s for new rows beyond the latest `id` at connect time. Optional filters narrow the stream to `plan_id` and/or `agent_role`.
* **Data Format**: `data: <JSON_STRING>\\n\\n`
* **Initial event**: `ready` with `{ "connected": true }`
* **Example Event**:
```text
data: {"type":"admission","payload":{"id":1043,"receipt_id":"rcpt_771231_admi","event_type":"admission","plan_id":"plan_nexus_001","agent_role":"code_executor_role","work_request_id":"wrk_812","payload":{"status":"ADMITTED","latency_ms":18},"replayed_at":null,"created_at":"2026-07-22T17:21:00.000Z"}}
```
* **Keepalive**: `: keepalive <timestamp>` comment every 15s.

### 3C. Transactions & Causal Graph

#### `GET /api/peb/transactions`
List execution transactions, newest first.
* **Query Params**: `entity_id`, `tool_name`, `admission_result`, `since` (ISO date), `limit` (default `100`, max `500`), `offset`
* **Response `200 OK`**:
```json
{
  "transactions": [
    {
      "id": "tx_994a_102",
      "idempotency_key": "key_abc",
      "entity_id": "agent:runner-pod-99",
      "admission_result": "ADMITTED",
      "tool_name": "execute_db_migration",
      "input": {},
      "output": {},
      "before_hash": "abc123",
      "after_hash": "def456",
      "state_delta": {},
      "created_at": "2026-07-22T17:10:00.000Z",
      "committed_at": "2026-07-22T17:10:01.000Z",
      "kernel_event_id": null,
      "kernel_event_type": null
    }
  ]
}
```

#### `GET /api/peb/transactions/{id}`
Fetch a single transaction by ID.
* **Response `200 OK`**: `{ "transaction": { ... } }`
* **`404`**: Returns `{ "error": "transaction not found" }`.

#### `GET /api/peb/transactions/{id}/lineage`
Returns the complete causal lineage for a transaction in one payload.

* **Response `200 OK`**:
```json

  "traces": [
    { "id": "trace_001", "transaction_id": "tx_994a_102", "parent_trace_id": null, "stage": "execute", "confidence": 0.98, "rejected_alternatives": [], "status": "completed", ... }
  ],
  "traces_tree": [
    { "id": "trace_001", "children": [], "stage": "execute", "confidence": 0.98, ... }

    { "id": 1042, "receipt_id": "rcpt_889201", "event_type": "violation", "plan_id": "plan_nexus_002", ... }
```

### 3D. Causal Graph: Decisions & Traces

#### `GET /api/peb/decisions/{id}/chain?direction=ancestry`
Walks decision parent linkage (`parent_decision_id`) or rollback chain (`rollback_of`).
* **Query Params**: `direction`: `ancestry` (default) | `rollback`
* **Response `200 OK`**:
```json
{
  "direction": "ancestry",
  "chain": [
    { "id": "dec_8812", "parent_decision_id": "dec_8800", "title": "ADR-004", "status": "approved", "depth": 0, "created_at": "..." },
    { "id": "dec_8800", "parent_decision_id": null, "title": "ADR-003", "status": "approved", "depth": 1, "created_at": "..." }
  ]
}
```
* **`404`**: Returns `{ "error": "decision not found" }`.

#### `GET /api/peb/traces/{id}/tree`
Returns hierarchical trace node tree with `rejected_alternatives` and `confidence` at each node. Walks `parent_trace_id` recursively up to 200 levels.
* **Response `200 OK`**:
```json
{
  "root_id": "trace_001",
  "node_count": 5,
  "tree": [
    {
      "id": "trace_001",
      "transaction_id": "tx_994a_102",
      "parent_trace_id": null,
      "stage": "execute",
      "confidence": 0.98,
      "rejected_alternatives": [],
      "status": "completed",
      "depth": 0,
      "children": [
        {
          "id": "trace_002",
          "parent_trace_id": "trace_001",
          "stage": "validate",
          "confidence": 0.95,
          "rejected_alternatives": [{ "stage": "validate_alt", "reason": "slower path" }],
          "children": [],
          ...
        }
      ]
    }
  ]
}
```
* **`404`**: Returns `{ "error": "trace not found" }`.

### 3E. Capability Gap Analysis

#### `GET /api/peb/entities/{entity_id}/capability-gap`
Overlay analysis of capability grants vs attempted violations for an agent entity. For each violation, checks whether the entity held an active grant at the moment of the attempt.

* **Query Params**: `limit` (default: 100), `offset` (default: 0)
* **Response `200 OK`**:
```json
{
  "entity_id": "agent:runner-pod-99",
  "capability_gaps": [
    {
      "violation_id": "v_005",
      "violation_type": "UNAUTHORIZED_CAPABILITY_ACCESS",
      "severity": "CRITICAL",
      "capability_attempted": "exec:raw_sql",
      "context": { "query": "DROP TABLE..." },
      "resolution": null,
      "violation_created_at": "2026-07-22T17:15:30.000Z",
      "active_grants_at_violation": [],
      "lapsed_grants_at_violation": [
        {
          "capability_id": "cap_001",
          "capability": "exec:raw_sql",
          "granted_by": "admin",
          "granted_at": "2026-07-01T00:00:00Z",
          "expires_at": "2026-07-15T00:00:00Z",
          "active": false
        }
      ],
      "gap_status": "lapsed"
    }
  ],
  "summary": {
    "active": 5,
    "lapsed": 2,
    "missing": 3
  }
}
```

**`gap_status` values:**
- `active` — one or more active grants existed at the moment the attempt was made (the violation is a deliberate overstep)
- `lapsed` — a capability grant with the same name existed for this entity once but had expired or been deactivated before this attempt
- `missing` — no capability grant ever existed for this entity + capability

#### `GET /api/peb/entities/{entity_id}/capabilities`
Convenience endpoint: list all capability grants for an entity.
* **Response `200 OK`**:
```json
{
  "entity_id": "agent:runner-pod-99",
  "capabilities": [
    {
      "id": "cap_002",
      "entity_id": "agent:runner-pod-99",
      "capability": "exec_container_script",
      "granted_by": "admin",
      "expires_at": null,
      "active": true,
      "created_at": "2026-07-01T00:00:00Z",
      "status": "active"
    }
  ]
}
```

### 3F. State Delta & Versioning

#### `GET /api/peb/state/{key}/versions`
Lists all transactions that touched a state key (derived from `transactions.state_delta`), plus the current `peb.state` row.
* **Response `200 OK`**:
```json
{
  "key": "governance.policy.v2",
  "current": {
    "id": "state_001",
    "key": "governance.policy.v2",
    "content": { "mode": "STRICT", "max_memory_mb": 2048 },
    "metadata": {},
    "checksum": "abc123",
    "version": 5,
    "created_at": "...",
    "updated_at": "..."
  },
  "historical_versions": [
    { "transaction_id": "tx_994a_080", "created_at": "...", "committed_at": "...", "touched_key": true },
    { "transaction_id": "tx_994a_102", "created_at": "...", "committed_at": "...", "touched_key": true }
  ],
  "version_count": 3
}
```

#### `GET /api/peb/state/{key}/diff?from=<tx_id>&to=<tx_id|current>`
Replays `state_delta` patches across transactions that touched the key, reconstructs snapshots at two points, and returns the field-level diff.
* **Query Params**:
  * `from` (required): transaction ID for the "before" side
  * `to` (required): transaction ID for the "after" side, or the special value `current` to use the live `peb.state` row
* **Response `200 OK`**:
```json
{
  "key": "governance.policy.v2",
  "from": {
    "transaction_id": "tx_994a_080",
    "content": { "mode": "EVALUATION", "max_memory_mb": 1024 }
  },
  "to": {
    "transaction_id": null,
    "content": { "mode": "STRICT", "max_memory_mb": 2048 }
  },
  "diff": {
    "added": {},
    "removed": [],
    "changed": [
      { "key": "mode", "from": "EVALUATION", "to": "STRICT" },
      { "key": "max_memory_mb", "from": 1024, "to": 2048 }
    ]
  }
}
```

## 4. Database Schema (PEB — No FK Constraints)

To support high-throughput event sourcing without foreign key locks, `peb-srv` uses 8 tables in the `peb` schema with **no foreign-key constraints** (joins are logical and encoded in the REST layer):

| Table | Purpose |
|:---|:---|
| `peb.governance_events` | Event ledger — `id` (bigserial) serves as streaming cursor, `receipt_id` is the external event reference |
| `peb.role_circuit_breaker` | Safety circuit breaker state per agent role |
| `peb.violations` | Audit records with `entity_id`, `violation_type`, `severity`, `capability_attempted` |
| `peb.capabilities` | Capability grants per entity with `active`, `granted_by`, `expires_at` |
| `peb.transactions` | Tool executions with `state_delta` (jsonb patch) for state history reconstruction |
| `peb.decisions` | Causal graph — `parent_decision_id`, `rollback_of`, `entropy_class` |
| `peb.traces` | Execution trees — `parent_trace_id`, `rejected_alternatives`, `confidence` |
| `peb.state` | Current-value table — `key`, `content` (jsonb), `version`, `checksum` |

**State history note:** `peb.state` holds **current values only**. The historical record is reconstructed from `peb.transactions.state_delta` by replaying patches in chronological order.

## 5. Testing Your Live API Integration

1. Start your `peb-srv` backend on port `4206` (or your chosen port).
2. Open the NEXUS UI in your browser.
3. Click the **"MOCK ENGINE"** button in the header (or navigate to **"API Specs"** tab and click **"CONFIGURE MOCK SCHEME"**).
4. Set **API Mode** to `LIVE API MODE` and verify the Base URL (`http://localhost:4206`).
5. Click **"Test Connection"** to verify `/health` returns `200 OK`.
6. Click **"Save Configuration"** — the dashboard will now consume live data and SSE streams.

## 6. Verification `curl` Commands

```bash
# Health check
curl -X GET http://localhost:4206/health

# Fetch audit events (cursor-based, limit 5)
curl -X GET "http://localhost:4206/api/peb/events?limit=5"

# Fetch events since cursor 1000
curl -X GET "http://localhost:4206/api/peb/events?since=1000&limit=20"

# Fetch single event by receipt ID
curl -X GET http://localhost:4206/api/peb/events/rcpt_1001

# Trigger replay on event
curl -X POST http://localhost:4206/api/peb/events/rcpt_1001/replay

# List transactions
curl -X GET "http://localhost:4206/api/peb/transactions?limit=5"

# Transaction lineage
curl -X GET http://localhost:4206/api/peb/transactions/tx_994a_102/lineage

# Decision chain (ancestry)
curl -X GET "http://localhost:4206/api/peb/decisions/dec_8812/chain?direction=ancestry"

# Trace tree
curl -X GET http://localhost:4206/api/peb/traces/trace_001/tree

# Capability gap for an entity
curl -X GET http://localhost:4206/api/peb/entities/agent:runner-pod-99/capability-gap

# Entity capabilities
curl -X GET http://localhost:4206/api/peb/entities/agent:runner-pod-99/capabilities

# Circuit breakers
curl -X GET http://localhost:4206/api/peb/health/circuit-breakers

# Violations summary (24h, grouped by severity)
curl -X GET "http://localhost:4206/api/peb/health/violations/summary?window=24h&group_by=severity"

# Entropy rollup
curl -X GET "http://localhost:4206/api/peb/health/entropy?group_by=entropy_class&window=14d"

# State versions for a key
curl -X GET http://localhost:4206/api/peb/state/governance.policy.v2/versions

# State diff between two transactions
curl -X GET "http://localhost:4206/api/peb/state/governance.policy.v2/diff?from=tx_994a_080&to=tx_994a_102"

# State diff to current
curl -X GET "http://localhost:4206/api/peb/state/governance.policy.v2/diff?from=tx_994a_080&to=current"

# Listen to SSE stream
curl -N http://localhost:4206/api/peb/events/stream
```
