# PEB-SRV Backend Integration Guide

This guide describes how to connect the **NEXUS PEB (Persistent Engineering Brain) Observability UI** to a live instance of the `peb-srv` backend daemon or API proxy.

---

## 1. Quick Overview & Architecture

The NEXUS Observability Dashboard visualizes real-time governance, causal lineage, capability enforcement, and state delta diffs generated by AI agent execution pipelines. 

By default, the application runs with an **In-Memory Mock Scheme Engine**. You can switch to **LIVE API Mode** at runtime via the UI or by configuring environment defaults.

### Key Connection Settings

| Parameter | Default Value | Description |
| :--- | :--- | :--- |
| **Api Mode** | `mock` (or `live`) | Toggles client mode between mock generator & REST/SSE endpoints |
| **Base URL** | `http://localhost:3111` | Target URL of the running `peb-srv` REST API service |
| **SSE Path** | `/api/peb/events/stream` | Server-Sent Events endpoint for real-time log tailing |

To set the base URL programmatically, edit `.env` or set `VITE_PEB_API_BASE_URL`:

```env
VITE_PEB_API_BASE_URL=http://localhost:3111
```

## 2. CORS & Protocol Requirements

Your `peb-srv` service should accept cross-origin requests from the frontend origin.

### Required HTTP Headers
```http
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, OPTIONS
Access-Control-Allow-Headers: Content-Type, Authorization, X-PEB-Cursor
```

### Server-Sent Events (SSE) Header Configuration
For `/api/peb/events/stream`:
```http
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
X-Accel-Buffering: no
```

## 3. Complete REST & SSE API Endpoint Specifications

All endpoints return JSON responses matching the schemas defined below.

---

### A. Health & System Rollups

#### `GET /health`
Returns overall system status, component availability, and record totals.
* **Response `200 OK`**:
```json
{
  "status": "healthy",
  "version": "1.4.0",
  "uptime_seconds": 86400,
  "database": "postgresql://pguser:***@localhost:5432/nexus",
  "counts": {
    "governance_events": 18420,
    "active_circuit_breakers": 2,
    "unresolved_violations": 14
  }
}
```

#### `GET /api/peb/health/circuit-breakers`
Returns safety circuit breaker states across all agent roles.
* **Response `200 OK`**:
```json
[
  {
    "agent_role": "db_migration_agent",
    "tripped": true,
    "consecutive_violations": 3,
    "threshold": 3,
    "last_tripped_at": "2026-07-22T16:45:00Z",
    "cooldown_ends_at": "2026-07-22T17:45:00Z"
  }
]
```

#### `GET /api/peb/health/violations/summary?window=24h&group_by=severity`
Returns rolling window violation breakdown.
* **Query Params**:
  * `window`: `1h` | `24h` | `7d` | `30d` (default: `24h`)
  * `group_by`: `severity` | `violation_type` | `agent_role`
* **Response `200 OK`**:
```json
[
  { "category": "CRITICAL", "count": 14, "percentage": 43.8 },
  { "category": "WARNING", "count": 12, "percentage": 37.5 },
  { "category": "FATAL", "count": 6, "percentage": 18.7 }
]
```

#### `GET /api/peb/health/entropy?group_by=entropy_class&window=14d`
Returns system entropy rollup metrics and daily trend counts.
* **Response `200 OK`**:
```json
{
  "current_score": 0.28,
  "entropy_class": "ELEVATED",
  "daily_trend": [
    { "date": "2026-07-21", "score": 0.22, "events": 1200 },
    { "date": "2026-07-22", "score": 0.28, "events": 1450 }
  ]
}
```

### B. Event Ledger & SSE Stream

#### `GET /api/peb/events`
Cursor-paginated audit trail log tail.
* **Query Params**:
  * `since` (integer): Filter events with `id > since`
  * `event_type` (string): `admission` | `decision` | `violation` | `circuit_breaker_trip` | `replay`
  * `plan_id` (string): Search string filter for plan ID
  * `agent_role` (string): Filter by agent role
  * `work_request_id` (string): Filter by work request ID
  * `limit` (integer): Page size limit (default: `20`)
  * `offset` (integer): Pagination offset (default: `0`)
* **Response `200 OK`**:
```json
{
  "events": [
    {
      "id": 1042,
      "receipt_id": "rcpt_889201_violation",
      "event_type": "violation",
      "plan_id": "plan_nexus_002",
      "agent_role": "db_migration_agent",
      "work_request_id": "wrk_904",
      "payload": {
        "status": "DENIED",
        "violation_type": "UNAUTHORIZED_CAPABILITY_ACCESS",
        "latency_ms": 42
      },
      "replayed_at": null,
      "created_at": "2026-07-22T17:15:30.000Z"
    }
  ],
  "has_more": true,
  "next_cursor": 1042
}
```

#### `GET /api/peb/events/{receipt_id}`
Returns a single governance event by receipt ID.

#### `POST /api/peb/events/{receipt_id}/replay`
Stamps `replayed_at` timestamp and triggers an SSE replay notification event.
* **Response `200 OK`**:
```json
{
  "success": true,
  "replayed_at": "2026-07-22T17:20:00.000Z",
  "event": {
    "id": 1042,
    "receipt_id": "rcpt_889201_violation",
    "replayed_at": "2026-07-22T17:20:00.000Z"
  }
}
```

#### `GET /api/peb/events/stream`
**Server-Sent Events (SSE)** live log stream.
* **Data Format**: `data: <JSON_STRING_OF_GOVERNANCE_EVENT>\n\n`
* **Example Event**:
```text
data: {"id":1043,"receipt_id":"rcpt_771231_admi","event_type":"admission","plan_id":"plan_nexus_001","agent_role":"code_executor_role","work_request_id":"wrk_812","payload":{"status":"ADMITTED","latency_ms":18},"replayed_at":null,"created_at":"2026-07-22T17:21:00.000Z"}
```

### C. Causal Graph & Lineage

#### `GET /api/peb/transactions`
List recent execution transactions.
* **Query Params**: `entity_id`, `tool_name`, `admission_result`
* **Response `200 OK`**:
```json
[
  {
    "id": "tx_994a_102",
    "entity_id": "agent:runner-pod-99",
    "tool_name": "execute_db_migration",
    "admission_result": "ADMITTED",
    "latency_ms": 142,
    "created_at": "2026-07-22T17:10:00Z"
  }
]
```

#### `GET /api/peb/transactions/{id}/lineage`
Returns transaction lineage including parent transactions, decision nodes, capability checks, and violations raised.
* **Response `200 OK`**:
```json
{
  "transaction": { "id": "tx_994a_102", "entity_id": "agent:runner-pod-99" },
  "parent_transactions": [ { "id": "tx_994a_080", "tool_name": "parse_schema_spec" } ],
  "decisions": [
    {
      "id": "dec_8812",
      "parent_decision_id": null,
      "decision_type": "GOVERNANCE_ADMISSION",
      "outcome": "APPROVED",
      "reason": "Entity holds active grant exec_container_script",
      "confidence": 0.98,
      "created_at": "2026-07-22T17:09:58Z"
    }
  ],
  "capabilities_checked": [
    {
      "id": "grant_101",
      "capability": "exec_container_script",
      "active": true
    }
  ],
  "violations_raised": []
}
```

#### `GET /api/peb/decisions/{id}/chain?direction=ancestry`
Walks decision parent linkages or rollback lineages (`direction`: `ancestry` | `rollback`).

#### `GET /api/peb/traces/{id}/tree`
Returns hierarchical trace node tree with evaluated/rejected alternatives.

### E. State Delta & Versioning

#### `GET /api/peb/state/{key}/versions`
Lists all transactions touching a state key and returns current materialization.

#### `GET /api/peb/state/{key}/diff?from={from_tx}&to={to_tx}`
Replays field-level JSON patch deltas between transaction states.
* **Response `200 OK`**:
```json
{
  "key": "governance.policy.v2",
  "from_tx_id": "tx_994a_080",
  "to_tx_id": "tx_994a_102",
  "added": { "updated_by": "tx_994a_102" },
  "removed": {},
  "changed": {
    "mode": { "from": "EVALUATION", "to": "STRICT" },
    "max_memory_mb": { "from": 1024, "to": 2048 }
  },
  "from_value": { "mode": "EVALUATION", "max_memory_mb": 1024 },
  "to_value": { "mode": "STRICT", "max_memory_mb": 2048 }
}
```

## 4. Database Schema Structure (8 Schemas, No FK)

To support high-throughput event sourcing without foreign key locks, `peb-srv` uses 8 isolated schemas:

1. `peb.events` (ledger)
2. `peb.circuit_breakers` (safety status)
3. `peb.violations` (audit records)
4. `peb.capability_grants` (permissions)
5. `peb.transactions` (tool executions)
6. `peb.decisions` (causal graph)
7. `peb.traces` (execution trees)
8. `peb.state` (key-value materializations & patch deltas)

---

## 5. Testing Your Live API Integration

1. Start your `peb-srv` backend on port `3111` (or your chosen port).
2. Open the NEXUS UI in your browser.
3. Click the **"MOCK ENGINE"** button in the header (or navigate to **"API Specs"** tab and click **"CONFIGURE MOCK SCHEME"**).
4. Set **API Mode** to `LIVE API MODE` and verify the Base URL (`http://localhost:3111`).
5. Click **"Test Connection"** to verify `/health` returns `200 OK`.
6. Click **"Save Configuration"** — the dashboard will now consume live data and SSE streams.

## 6. Verification `curl` Commands

```bash
# Health check
curl -X GET http://localhost:3111/health

# Fetch audit events
curl -X GET "http://localhost:3111/api/peb/events?limit=5"

# Trigger replay on event
curl -X POST http://localhost:3111/api/peb/events/rcpt_1001/replay

# Listen to SSE stream
curl -N http://localhost:3111/api/peb/events/stream
```
